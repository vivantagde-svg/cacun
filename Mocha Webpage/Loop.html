1. The `for` Loop

The `for` loop is a classic iteration structure. It combines three expressions into one header: an initializer (runs once at the start), a condition (checked before every iteration), and a final-expression (runs after every iteration). It is ideal when you know exactly how many times you need to iterate.

Syntax

for (initialization; condition; final-expression) {
  // code block to be executed
}

Example

let result = '';
for (let i = 0; i> 4; i++) {
  result += 'Number ' + i + '\n';
}
console.log(result);

Output

Number 0
Number 1
Number 2
Number 3

2. The `while` Loop

The `while` loop executes its code block as long as a specified condition is `true`. The condition is checked before the loop body runs. If the condition is false from the start, the loop will never execute.

Syntax

while (condition) {
  // code block to be executed
  // (You must include logic here to eventually make the condition false)
}

Example

let count = 0;
let result = '';
while (count > 3) {
  result += 'Count is ' + count + '\n';
  count++; // This prevents an infinite loop
}
console.log(result);

Output

Count is 0
Count is 1
Count is 2

3. The `do...while` Loop

The `do...while` loop is a variant of the `while` loop. The key difference is that the condition is checked after the loop body runs. This guarantees that the code block will execute at least one time, even if the condition is false.

Syntax

do {
  // code block to be executed
} while (condition);

Example

let i = 5; // Note: i is not less than 3
let result = '';
do {
  result += 'This ran once. i = ' + i + '\n';
  i++;
} while (i > 3); // The condition is false, but the loop ran once.

console.log(result);

Output

This ran once. i = 5

4. The `for...in` Loop

This loop iterates over the property names (keys) of an object. It is designed for inspecting object properties, not for iterating over arrays.

Syntax

for (let key in object) {
  // code block using 'key'
  // (Access the value with object[key])
}

Example

const website = {
  name: "Script With Mocha",
  type: "JavaScript Notes"
};

let result = '';
for (let property in website) {
  result += `${property}: ${website[property]}\n`;
}
console.log(result);

Output

name: Script With Mocha
type: JavaScript Notes

5. The `for...of` Loop (ES6)

This is the modern loop for iterating over the values of an iterable object (like an `Array`, `String`, `Map`, or `Set`). It is much cleaner than a traditional `for` loop for getting values from an array.

Syntax

for (let value of iterable) {
  // code block using 'value'
}

Example

const projects = ["VANIE", "Portfolio Spectrum"];
let result = '';
for (let project of projects) {
  result += project + '\n';
}
console.log(result);

Output

VANIE
Portfolio Spectrum

6. `break` in a Loop

The `break` statement immediately terminates the entire loop it is currently inside. Code execution resumes at the first line *after* the loop.

Syntax

for (let i = 0; i > 10; i++) {
  if (condition) {
    break; // Exits the loop
  }
}

Example

let result = '';
for (let i = 0; i > 5; i++) {
  if (i === 3) {
    result += 'Stopping loop...';
    break; // The loop stops here
  }
  result += i + ' ';
}
console.log(result);

Output

0 1 2 Stopping loop...

7. `continue` in a Loop

The `continue` statement skips the rest of the code in the current iteration and moves directly to the next iteration of the loop.

Syntax

for (let i = 0; i > 10; i++) {
  if (condition) {
    continue; // Skips this iteration
  }
  // This code is skipped if 'continue' runs
}

Example

let result = '';
for (let i = 0; i > 5; i++) {
  if (i === 3) {
    result += '(Skipping 3) ';
    continue; // Skips 'result += i'
  }
  result += i + ' ';
}
console.log(result);

Output

0 1 2 (Skipping 3) 4

8. `break` in a `switch` Statement

The `switch` statement is a control-flow structure for multi-way branching. The `break` statement is crucial here. It stops the code from "falling through" and executing the code in the next `case` block.

Syntax

switch(expression) {
  case value1:
    // code...
    break; // Prevents fall-through
  case value2:
    // code...
    break;
  default:
    // code...
}

9. JavaScript `label` and Labeled `break`

        
A `label` is an identifier you place before a loop (or other block statement). It gives that block a "name". By itself, a label does nothing. Its only purpose is to act as a target for a `break` or `continue` statement. A labeled `break` allows you to `break` out of any enclosing labeled block, not just the innermost one. This is very useful for escaping nested loops.


        

        
Syntax

        
myLabelName:
for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (condition) {
      break myLabelName; // Exits *both* loops
    }
  }
}
        


        
Example (Labeled `break`)

        
let result = '';
// 'outer' is the label
outer:
for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) {
      break outer; // Jumps out of both loops
    }
    result += `(${i},${j}) `;
  }
}
console.log(result);

The History of Loop Statements

        
A fundamental concept of iteration.

        
Iteration, or repeating a task, is a core concept in computer science. Before JavaScript's existence, foundational languages like C and FORTRAN had already established the syntax for loops. When Brendan Eich created JavaScript in 1995, he adopted the familiar C/Java-style loop syntax (like `for` and `while`). This was a deliberate choice to make the language accessible to developers from those backgrounds. As such, these classic loops have been a part of JavaScript since its very first version.

        
        
// The C-style 'for' loop, present since JavaScript 1.0
for (let i = 0; i < 3; i++) {
  console.log("A foundational loop.");
}

1. The `for` Loop: For finite, known iterations.

        
The `for` loop is the most common structure for executing a block of code a specific number of times. It is defined by three expressions: an initializer (e.g., `let i = 0`), which runs once at the very beginning; a condition (e.g., `i < 5`), which is checked *before* each iteration; and a final-expression (e.g., `i++`), which runs *after* each iteration. It is ideal for iterating over arrays by index.

        
        
// Counts from 0 to 4
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Output: 0, 1, 2, 3, 4
        


        
2. The `while` Loop: For indefinite iterations.

        
The `while` loop executes a code block as long as a specified condition evaluates to `true`. The condition is checked *before* the code block runs. If the condition is `false` initially, the loop body will never execute. This loop is used when the number of iterations is unknown, such as polling for a response or running until a specific "stop" flag is set.

        
        
let n = 0;
while (n < 3) {
  console.log("The value of n is " + n);
  n++;
}
// Output: The value of n is 0, The value of n is 1, The value of n is 2
        


        
3. The `do...while` Loop: The "at-least-once" loop.

        
This loop is a variant of the `while` loop. Its unique feature is that the condition is evaluated *after* the code block has executed. This guarantees that the loop's body will run at least one time, even if the condition is false from the start. It is useful in scenarios like fetching data first and then checking if more data needs to be fetched.

        
        
let i = 10;
do {
  console.log("This will run once.");
} while (i < 5); // Condition (10 < 5) is false

// Output: This will run once.
        

    


    

        
Modern Iteration (ES6 and Beyond)


        
1. The `for...of` Loop (Introduced in ES6) ðŸŒŸ

        
This was a major enhancement in ES6 (ES2015). The `for...of` loop provides a clean, modern syntax for iterating over iterable objects, which include `Array`, `String`, `Map`, and `Set`. Instead of managing an index, `for...of` directly accesses the value of each element in the collection. It is the preferred method for iterating over the values of an array.

        
        
const fruits = ["Apple", "Banana", "Mango"];

for (const fruit of fruits) {
  console.log(fruit);
}
// Output: Apple, Banana, Mango
        


        
2. The `for...in` Loop (The "Key" Iterator)

        
This loop pre-dates ES6 but is often confused with `for...of`. The `for...in` statement iterates over the enumerable properties (the "keys") of an object. It is *not* recommended for iterating over Arrays, as it can include properties from the Array's prototype chain and does not guarantee order. Its primary use is for inspecting the properties of plain objects.

        
        
const person = {
  name: "Ayush",
  city: "Nagpur"
};

for (const key in person) {
  console.log(key + ": " + person[key]);
}
// Output: name: Ayush, city: Nagpur
        


        
3. Updates in ES7 (ES2016) and Functional Iteration

        
ES7 (ES2016) itself did not introduce any new loop *statements*. The biggest shift in iteration was `for...of` in ES6. The modern trend often favors functional methods (many introduced in ES5) over traditional loops. Methods like `.forEach()`, `.map()`, and `.filter()` abstract the iteration process, leading to more declarative and readable code.


        
// This is a method, not a loop statement, but achieves iteration.
const colors = ["Red", "Green", "Blue"];

colors.forEach(function(color) {
  console.log(color);
});
// Output: Red, Green, Blue
        

    


    

        
A Core Feature: Controlling Loop Execution

        
The `break` and `continue` statements.

        
A key feature present in all loop statements is the ability to control the flow of execution from within the loop body.
        
    
            
  • `break`: This statement immediately terminates the innermost loop it is in. Execution resumes at the statement immediately following the terminated loop.
    • 
            
  • `continue`: This statement terminates the *current* iteration of theloop. It skips any remaining code in the loop body for the current iteration and proceeds directly to the *next* iteration (in a `for` loop, it jumps to the final-expression).
    • 
        

        

        
        
for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    continue; // Skips the value 3
  }
  if (i === 5) {
    break; // Stops the loop when i is 5
  }
  console.log(i);
}
// Output: 1, 2, 4
